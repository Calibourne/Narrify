# Narrify UI — Design Spec

**Date:** 2026-05-01  
**Status:** Approved

---

## Context

The parsing layer (`POST /api/parse`) is complete. This spec covers the frontend UI that lets users upload a book file and browse the parsed chapters. The layout is designed to scale cleanly when audio dubbing (edge-tts) is added later.

---

## Layout

**Split panel** with a top nav bar.

```
┌─────────────────────────────────────────────┐
│  Narrify                          ☀ / 🌙    │  ← nav bar
├──────────────┬──────────────────────────────┤
│              │                              │
│  LEFT PANEL  │  RIGHT PANEL                 │
│  (controls)  │  (chapters)                  │
│              │                              │
└──────────────┴──────────────────────────────┘
```

On mobile (< 640px): panels stack vertically — left panel on top, right panel below.

---

## Nav Bar

- Left: "Narrify" wordmark
- Right: `ThemeToggle` button (☀ / 🌙 label)
- Fixed height, no shadow — clean and minimal

---

## Left Panel

### Phase 1 (current scope)

1. **Section label:** "Book" (small uppercase, muted)
2. **`UploadZone`** — drag-and-drop area + hidden `<input type="file" accept=".epub,.fb2">`. Click anywhere on the zone opens the file picker. Drag-and-drop as bonus.
3. **File name display** — shown after a file is selected. Filename + a clear (✕) button to reset.
4. **"Parse Book" button** — disabled until a file is selected. Shows a spinner while the API call is in flight.
5. **Inline error** — shown below the button if the API returns an error. Clears when a new file is selected.

### Phase 2 (future — audio dubbing, not in scope now)

The left panel will gain these sections below the upload block, without restructuring:
- Voice picker (language + voice name dropdowns)
- Speed slider (0.5× – 2×)
- "Generate Audio" button with progress bar
- Stats (`StatsBadge`)

For Phase 1, `StatsBadge` appears in the **right panel** (above the chapter list).

---

## Right Panel

### Empty state

Centered placeholder text: "Upload a book to get started." Shown until a successful parse.

### After parse

1. **`StatsBadge`** — "12 chapters · 847 paragraphs" in a single muted line
2. **`ChapterList`** — vertical accordion
   - First chapter expanded on load
   - All others collapsed
   - Click to toggle expand/collapse
3. **`ChapterItem`** (expanded):
   - Chapter title (bold) + collapse arrow
   - All paragraphs rendered as `<p>` tags
4. **`ChapterItem`** (collapsed):
   - Chapter title + expand arrow
   - No content visible
5. Chapter status badges (`✓ ready`, `⟳ generating`, dimmed/pending) are **reserved for the audio phase** — not rendered in Phase 1.

## Theme

- Two modes: **light** (default) and **dark**
- Toggled by `ThemeToggle`, persisted to `localStorage`
- Implemented via a `data-theme` attribute on `<html>`, with CSS custom properties in `globals.css`
- Existing vars (`--background`, `--foreground`) extended with surface, border, and muted-text tokens

### Color tokens (additions to globals.css)

| Token | Light | Dark |
|---|---|---|
| `--surface` | `#ffffff` | `#161b22` |
| `--surface-subtle` | `#f7f7f7` | `#0d1117` |
| `--border` | `#e0e0e0` | `#30363d` |
| `--text-muted` | `#888888` | `#8b949e` |
| `--accent` | `#111111` | `#58a6ff` |

## State (in `page.tsx`)

```ts
type Status = 'idle' | 'uploading' | 'success' | 'error'

const [status, setStatus] = useState<Status>('idle')
const [file, setFile] = useState<File | null>(null)
const [chapters, setChapters] = useState<Chapter[]>([])
const [errorMsg, setErrorMsg] = useState<string | null>(null)
const [theme, setTheme] = useState<'light' | 'dark'>('light')
```

No external state library needed. All state lives in `page.tsx` and is passed as props.

---

## Components

All in `src/components/`. No external UI library — vanilla CSS modules to match existing project setup.

| Component | Props | Responsibility |
|---|---|---|
| `ThemeToggle` | `theme`, `onToggle` | Renders ☀/🌙 button, applies `data-theme` to `<html>` |
| `UploadZone` | `onFile`, `disabled` | Drag-and-drop zone + file input |
| `StatsBadge` | `chapters: Chapter[]` | Renders "X chapters · Y paragraphs" |
| `ChapterList` | `chapters: Chapter[]` | Renders list of `ChapterItem`, first one open |
| `ChapterItem` | `chapter`, `defaultOpen` | Accordion item with title + paragraphs |

---

## Data Flow

```
User drops file
  → UploadZone calls onFile(file)
  → page.tsx sets file state, status = 'idle'

User clicks "Parse Book"
  → status = 'uploading'
  → POST /api/parse with FormData { file }
  → on success: status = 'success', chapters = response.chapters
  → on error: status = 'error', errorMsg = response.error
```

---

## Error Handling

| Scenario | UI response |
|---|---|
| Wrong file type selected | Browser-level (accept attr), no custom UI needed |
| API 400 (unsupported format) | Inline error below button |
| API 422 (malformed file) | Inline error below button |
| API 500 / network failure | Inline error: "Something went wrong. Try again." |
| File > 4.5MB | Inline warning before submit: "File may exceed server limit (4.5MB)" |

## Out of Scope (Phase 1)

- Audio generation / playback
- Voice picker, speed control
- File history or persistence
- Server-side rendering of chapters (client fetch only)
- Animations beyond CSS transitions on accordion
